JS中的this指向

一、

全局中调用this，那么this指向window

二、

  1. 在全局的函数中（两种情况）

    //第一种:window
    function f1(){

    console.log(this)

    }
    f1() //window

    //第二种：严格模式是undefined
    function f2(){

    'use strict'
console.log(this)

    }
    f2() //undefined

三、对象调用

函数中this在定义时是不能确定的，只有在执行时才知道，因为函数this永远指向最后调用这个函数的对象

let o1 = {
    a: 1,
    f3: function f3(){
        console.log(this)
    }
}

//1、用o1调用
o1.f3()   //{a: 1, f3: ƒ}

//2、不用o1
let f4 = o1.f3
f4()      //window

四、构造函数: 指向构造出的实例

一、正常情况：指向构造出的实例

function F(){
    this.b = 2
}
let f5 = new F()
console.log(f5.b)    //2, 所以f5上是有属性 b 的，也就是构造时this指向f5

二、非常情况（如果在构造函数有return）

  1. return 一个对象，那么构造实例就为这个对象，也就得不到正常结果，无所谓this指向了。

    //return 对象
function F(){
    this.b = 2
    let o2 = {name:'xiao'}    //xiao不加引号错误！
    return o2    
}
let f5 = new F()

console.log(f5)            //{name: "xiao"}
console.log(f5.b)        //undefined

  2. return 非对象，包括null，虽然 typeof(null) === ‘object’，那么新构造的实例还是正常的。

function F(){
    this.b = 2
    return null    //或者return 1/undefined/"abc"
}
let f6 = new F()

console.log(f6)   //{b:2}

到这里，我发现这些非常情况似乎已经不是this指向问题了，而是new的返回问题了。

所以记住，构造函数的this指向构造实例 就OK
